# 仕入れ候補発掘ツール

複数のECサイトから商品情報を収集し、**メルカリで販売した場合の利益を自動計算**して、
仕入れ候補の合否判定（◎○△×）付きでスプレッドシートに一覧化するCLIツールです。

設計書: [`../仕入れ候補発掘ツール設計書.md`](../仕入れ候補発掘ツール設計書.md)（v1.3準拠）

## セットアップ

```bash
cd sedori-tool
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt

# APIキーを設定（Phase 1 は楽天・Yahoo!の2つだけでOK。どちらも無料）
cp .env.example .env
# .env を編集して RAKUTEN_APP_ID / YAHOO_APP_ID を記入
```

- 楽天デベロッパー登録: https://webservice.rakuten.co.jp/
  - 新認証方式のため `RAKUTEN_APP_ID`（アプリケーションID・UUID形式）と
    `RAKUTEN_ACCESS_KEY`（アクセスキー・pk_...）の**両方**を .env に設定
  - アプリの「許可されたWebサイト」に実行元のグローバルIPを **CIDR形式**
    （例: `203.0.113.5/32`）で登録する。回線のIPが変わったら再登録が必要
    （現在のIPは `curl https://api.ipify.org` で確認できる）
- Yahoo!デベロッパー登録: https://e.developer.yahoo.co.jp/

## 使い方

### GUI（実行ボタン）で使う

Finderで **`実行ツール.command` をダブルクリック**すると、実行ボタン付きの画面が開きます。
（初回はセットアップが自動で走ります。macOSに「開発元が未確認」と言われた場合は
右クリック→「開く」で起動してください）

1. キーワードまたはJANコードを入力
2. 状態・件数・出力先を選んで「▶ 実行」
3. ログに進捗が流れ、完了後「📄 結果Excelを開く」で結果を確認

ターミナルからは `.venv/bin/python gui.py` でも起動できます。

### CLIで使う

```bash
# キーワード検索（有効な全サイト横断）
.venv/bin/python main.py search --keyword "ワイヤレスイヤホン" --max 20

# JANコード検索（特定商品の最安値比較・相場推定の精度が最も高い）
.venv/bin/python main.py search --jan 4901234567890

# サイト指定 / 中古のみ
.venv/bin/python main.py search --keyword "○○" --sites rakuten,yahoo --condition used

# 出力先の切り替え（デフォルトはExcel: 仕入れ候補.xlsx）
.venv/bin/python main.py search --keyword "○○" --output gsheet   # Google Sheets
.venv/bin/python main.py search --keyword "○○" --output csv      # CSV
```

## 運用フロー（設計書 7.2）

1. 検索すると、楽天/Yahoo!の同一JAN商品の中古相場中央値 × 補正係数0.85 で
   **想定販売価格を自動推定**し、損益を仮計算して ◎○△×/要目視 を判定
2. スプレッドシートの「相場確認URL」をクリック → メルカリの**売り切れ検索**が開く
3. 実勢価格を確認して「想定販売価格」列を上書き → **利益・判定が数式で自動再計算**
4. 「確定済み」列にチェックを付けてから仕入れ判断（推定値のまま仕入れないこと！）

## スプレッドシートの構成

| シート | 内容 |
|---|---|
| 候補一覧 | メイン出力。追記型・URL基準で重複スキップ。数式/リンク/プルダウン付き |
| 設定 | 手数料率・送料/資材代・判定基準（候補一覧の数式が参照。直接編集可） |
| 仕入れ済み | 仕入れた商品を手動で移動（在庫管理の入口） |
| 検索リンク | ヤフオク/ラクマ/Temu/Qoo10等（Tier3）の検索URL |

**設計書からの改善点**: 「輸入経費」列（J列）を追加し、海外仕入れの関税・輸入消費税の
概算を実質仕入れ価格の数式に含めています（設計書6.2の列は1つずつ後ろへずれます）。
また判定数式は、注意フラグに「輸入品」を含む行の利益率基準を自動で+5%（設定シートB2）
上乗せします。

## 対応サイト

| Tier | サイト | 状態 |
|---|---|---|
| 1（公式API） | 楽天市場 / Yahoo!ショッピング | ✅ キー設定のみで利用可（Phase 1） |
| 1（公式API） | Amazon PA-API / eBay / AliExpress | ✅ 実装済み。キー取得後 `config.yaml` の `sites.tier1` に追加（Phase 3） |
| 2（スクレイピング） | 駿河屋 | 実装済み。**規約確認の上** `tier2_scraping_enabled: true` で有効化。robots.txt を自動チェックし、不許可なら取得しない |
| 2 | ブックオフ / ハードオフ / ヨドバシ / セカンドストリート | Phase 4予定。当面は「検索リンク」シートでカバー |
| 3（URL生成） | ヤフオク / ラクマ / Temu / Qoo10 | ✅ 検索URLを自動生成 |

## 設定（config.yaml）

- メルカリ手数料率・送料/資材代テーブル・判定基準・輸入コスト概算・補正係数など、
  改定されうる数値はすべて `config.yaml` で一元管理
- 為替は無料API（open.er-api.com）から自動取得し、+3%のバッファを上乗せ。
  取得失敗時は `import.fallback_rates` の固定値にフォールバック

## 留意事項（設計書 11章より）

- **推定販売価格はあくまで足切り用**。仕入れ判断は必ずメルカリの売り切れ相場を目視確認後に
- 中古品を継続的に仕入れて販売する場合は**古物商許可**の要否を所轄警察署に確認
- 輸入品はPSE・技適・食品衛生法・薬機法等の規制対象カテゴリを避けること
- 輸入コストは概算。転売目的の輸入は個人使用の免税評価（60%）が適用されない場合あり
- スクレイピングは各サイトの利用規約を実装前に必ず確認（本ツールはrobots.txtの自動チェック付き）
